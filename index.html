React/Redux - Greco's Pizza Delivery

☲ View Site ☲ ☲ Github Repo ☲

This project is for an imaginary pizza parlour named Greco's. I wanted to make this when I originally learned how to build a website, I was going to ask one of the pizza places in town if they needed a website. However after looking, I saw that they had already had one. Anyway I decided to build the app as it would be a good project to put what I've learned into action as well as learn a few other things. This app uses a wizard Redux-Form to store the values of each order and submit them to the database. I implemented socket.io to update the admin page automatically whenever a new order comes in. This allows for the owner of the app to keep an updated view of orders awaiting to be delivered. Once the order is completed the admin would press the completed button to remove that order from the 'Open Orders' table on the admin page. One of the cells has a running count of the minutes since ordering, when the counter gets past 30 minutes the style is changed from green to red to indicate that the order is pushing peak delivery time.

To view in action open two tabs and navigate one tab to the admin page. Once you've submitted an order, it should automatically update on the admin side.

React - Weather Forecast App

☲ View Site ☲ ☲ Github Repo ☲

I built this as a project after finishing Tyler McGinnis' React Fundamentals course on his website. Basically it allows you to search for a city then makes an API call to openweathermap.org and returns the five day forecast for that specific city. You can click on each day to view the min and max temperatures as well as humidity and weather description.

React/Redux - UPC Inventory App

☲ View Site ☲ ☲ Github Repo ☲

This was the first project I completed upon finishing Stephen Grider's highly rated 'Modern React with Redux' course on Udemy. I developed it with the idea that business owner's would be able to use to fill out an online inventory for their store. What it does is it takes the UPC and makes an API call to upcitemdb.com and returns a bunch of different data which would save the business owner time in not having to find pictures or descriptions of the product. In it's current state it's not very user friendly as entering the UPC of each product would be more cumbersome than going and getting a description and images of the product. But my intentions are to come back to it in the future after learning mobile development to add a barcode scanner that can be used from a person's phone.

Express - littleBean

☲ View Site ☲ ☲ Github Repo ☲

This project was brought on by a friend asking if I could build him a website for his coffee distribution business. He didn't really give me much of a scope for the project so I went ahead and built something that I envisioned it working out as. I included authentication in this website along with email verification for newly created accounts. Inside the users' page I have tabs for open orders as well as completed orders that the customer can view. Also customers are able to edit their orders from the user page or delete them. There is a separate Order page for creating new orders. For the admin account, the users' page shows all of the open orders for the business, along with a completion button once the order has been filled. All in all I think this was a great learning experience that built confidence in my abilities with the MEN stack!
